「提案書を作るのに毎回時間がかかる」――フリーランスやコンサルタントとしてデータ分析サービスを提供していると、提案書の作成は避けて通れない作業です。

しかし、毎回ゼロから書くのは非効率です。特に、データ分析サービスの提案書は構成がある程度パターン化できるため、テンプレートを用意しておくと作業時間を大幅に短縮できます。

この記事では、実際に使っているデータ分析サービスの提案書テンプレート（Markdown形式）を公開し、Claude Codeを使ったカスタマイズ方法を紹介します。

## なぜMarkdownで提案書を書くのか

提案書のフォーマットは、PowerPoint、Googleスライド、PDFなどさまざまですが、自分はMarkdownで下書きを作成しています。理由は3つあります。

1. **テキストベースなので、バージョン管理しやすい**（Gitとの相性が良い）
2. **Claude Codeで直接編集・カスタマイズできる**
3. **最終的にPDFやスライドに変換可能**（Marpなどのツールを使用）

最終納品物はPDFやスライドにすることが多いですが、原稿をMarkdownで管理することで、テンプレートの再利用と更新が容易になります。

## 提案書テンプレート

以下が、データ分析サービスの提案書テンプレートです。クライアントの業種や課題に応じてカスタマイズして使います。

```markdown
# データ分析基盤構築のご提案

提出日: YYYY-MM-DD
提出先: {{クライアント名}} 御中
提出者: {{自社名}}

---

## 1. 背景と課題

{{クライアント名}}様では、現在以下のような課題を抱えていると
ヒアリングにて確認いたしました。

- {{課題1: 例）GA4を導入済みだが、データを経営判断に活用できていない}}
- {{課題2: 例）広告のチャネル別効果が把握できていない}}
- {{課題3: 例）月次レポートの作成に毎回多くの時間がかかっている}}

これらの課題は、データの収集はできているものの、
分析・可視化の仕組みが整備されていないことに起因しています。

---

## 2. ご提案内容

GA4の生データをBigQueryに蓄積し、
経営判断に活用できるダッシュボードを構築します。

### 構成概要

| レイヤー | 内容 | ツール |
|---------|------|--------|
| データ収集 | GA4イベントデータの自動エクスポート | GA4 → BigQuery |
| データ加工 | 生データのフラット化・集計ビューの構築 | BigQuery（SQL） |
| 可視化 | 経営ダッシュボードの構築 | Looker Studio |
| 自動化 | レポートの自動更新・アラート設定 | Cloud Scheduler |

### 納品物

1. BigQueryデータマート（3層設計: raw → staging → mart）
2. Looker Studioダッシュボード（{{ダッシュボード数}}画面）
3. 運用マニュアル（操作手順・トラブルシューティング）
4. 引き継ぎミーティング（1回、オンライン）

---

## 3. 期待される効果

### 広告効果の可視化
チャネル別・キャンペーン別のROASが可視化されることで、

手作業でのレポート作成が不要になります。

### 顧客行動の深い理解
ファネル分析、コホート分析により、
ユーザーの離脱ポイントやリピート状況が把握できます。
```

## Claude Codeでのカスタマイズ方法

このテンプレートをClaude Codeでカスタマイズする方法を紹介します。

### 基本的な使い方

テンプレートファイルを用意し、Claude Codeに以下のように指示します。

```
提案書テンプレート（proposal_template.md）を使って、
以下のクライアント向けにカスタマイズしてください。

- クライアント: 某美容系ECサイト
- 課題: 広告費のROASがわからない、カート離脱率が高い
- 予算感: 中程度
- ダッシュボード: 3画面（売上概要、チャネル別、ファネル）
```

Claude Codeはテンプレートの `{{}}` 部分を埋めるだけでなく、クライアントの業種や課題に合わせて文章のニュアンスも調整してくれます。

### 業種別のカスタマイズ

データ分析の提案は、クライアントの業種によって重点ポイントが変わります。

**EC事業者の場合:**
- ファネル分析（商品閲覧→カート→購入）を強調
- コホート分析（リピート率）の重要性を訴求
- チャネル別ROASの可視化を前面に出す

**SaaS事業者の場合:**
- ユーザーのアクティベーション率に焦点
- 解約率（チャーン）の分析を提案に含める
- MRR（月次定期収益）の可視化を提案

**メディア運営者の場合:**
- PV・UUの詳細分析
- 記事別のエンゲージメント分析
- 広告収益の最適化に焦点

Claude Codeに業種を伝えると、テンプレートの「期待される効果」セクションを業種に合わせて書き換えてくれます。

### 提案書の品質チェック

カスタマイズした提案書をClaude Codeにレビューしてもらうことも有効です。

```
以下の提案書をレビューしてください。
チェックポイント:
- クライアントの課題と提案内容の整合性
- 専門用語が多すぎないか
- 費用対効果の訴求が十分か
- 誇大表現がないか
```

提案書は技術的な正確さだけでなく、クライアントへの伝わりやすさも重要です。Claude Codeに「非エンジニアの経営者にも伝わるか」という観点でレビューしてもらうことで、提案の質を高められます。

## テンプレートの運用と改善

テンプレートは一度作ったら終わりではありません。案件を重ねるごとに改善していきます。

**改善サイクル:**

1. テンプレートを使って提案書を作成する
2. クライアントの反応をメモする（どの部分に興味を持ったか、どこで質問が出たか）
3. 反応が良かった表現はテンプレートに反映する
4. 質問が多かった部分は説明を追加する

このサイクルを回すことで、テンプレートの精度が上がり、受注率の向上につながります。

## 提案書以外への応用

このテンプレートの考え方は、提案書以外にも応用できます。

**月次レポートのテンプレート:**
クライアントへの月次レポートも、テンプレートを用意しておくと効率的です。数値の部分だけ毎月更新し、分析コメントをClaude Codeで生成するという運用が可能です。

**サービス説明資料:**
ココナラやポートフォリオサイトに掲載するサービス説明も、提案書のテンプレートをベースに作成できます。

**ヒアリングシート:**
提案書の「背景と課題」セクションは、クライアントへのヒアリング項目としてそのまま使えます。

## まとめ

提案書の作成は、フリーランスにとって受注を左右する重要な工程です。しかし、毎回ゼロから書く必要はありません。

テンプレートを用意し、Claude Codeでカスタマイズする仕組みを作ることで、提案書の作成時間を短縮しつつ、品質を維持できます。

今回公開したテンプレートは、自分の業務で実際に使っているものをベースにしています。ぜひカスタマイズして、自分のサービスに合った形で活用してみてください。
