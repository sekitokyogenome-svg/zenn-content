## はじめに

「Looker Studioのダッシュボードをスマホで見ようとしたら、文字が小さくて読めない」「横スクロールが必要で操作しにくい」という経験はないでしょうか。

経営者やマーケティング担当者は、移動中や外出先でスマホからダッシュボードを確認することが多いです。しかし、PC画面を前提に作られたダッシュボードは、そのままではスマホで使い物になりません。

この記事では、Looker Studioでスマホに最適化されたダッシュボードを設計・構築する具体的な手順を解説します。

## スマホ最適化の3つのアプローチ

Looker Studioでスマホ対応する方法は主に3つあります。

| アプローチ | 概要 | 手間 |
|---|---|---|
| レポートサイズの変更 | キャンバスサイズを縦長に変更 | 低 |
| 専用スマホページの追加 | PC用とスマホ用のページを分ける | 中 |
| レスポンシブレイアウト | 1つのレイアウトで両方に対応 | 高 |

現時点のLooker Studioにはレスポンシブ対応の自動機能がないため、「専用スマホページの追加」が実用的な選択肢です。

## 方法1: キャンバスサイズを縦長に設定する

### 設定手順

1. Looker Studioのレポート編集画面を開く
2. 「ファイル」→「レポートの設定」（またはキャンバス外をクリック）
3. 「レイアウト」タブで以下を設定

```
キャンバスサイズ:
  幅: 414px（iPhone標準）
  高さ: 2000px〜（コンテンツに応じて調整）
```

:::message
一般的なスマホの画面幅は375px〜428pxです。414pxはiPhoneの標準的な幅で、多くのAndroidデバイスにも対応できます。
:::

### 縦長レイアウトの設計ポイント

スマホでは横に並べず、縦に積み上げるレイアウトが基本です。

```
┌──────────────────┐
│ 日付フィルタ       │
├──────────────────┤
│ 売上スコアカード   │
├──────────────────┤
│ セッション         │
│ スコアカード       │
├──────────────────┤
│ CVR              │
│ スコアカード       │
├──────────────────┤
│ 売上推移          │
│ 折れ線グラフ      │
├──────────────────┤
│ チャネル別売上     │
│ 棒グラフ          │
├──────────────────┤
│ 商品TOP10        │
│ テーブル          │
└──────────────────┘
```

## 方法2: PC用とスマホ用のページを分ける（推奨）

1つのレポート内に複数のページを作り、PC用ページとスマホ用ページを分けるのが最も実用的です。

### 実装手順

1. 既存のPCレイアウト（幅1200px程度）のページはそのまま残す
2. 新しいページを追加（「ページ」→「新しいページ」）
3. 新しいページのキャンバスサイズを `414 x 2000` に設定
4. スマホ用のグラフを配置する

### スマホページの命名規則

ページ名を「(SP) ダッシュボード」「(PC) ダッシュボード」のように分けておくと、閲覧者がわかりやすくなります。

### スマホページ用のURL共有

Looker StudioのURLにページ番号パラメータを追加すると、特定のページを直接開けます。

```
https://lookerstudio.google.com/reporting/REPORT_ID/page/PAGE_ID
```

スマホページのURLをブックマークやショートカットに登録してもらえば、毎回PC用ページを経由する手間がなくなります。

## スマホ用グラフの設計ガイドライン

### スコアカードを中心に構成する

スマホではグラフの細部が見にくいため、スコアカード（数値表示）を中心に構成します。

```
┌──────────┬──────────┐
│ 売上      │ 注文数    │
│ ¥1,234K  │ 156件    │
│ +12.3%   │ +8.5%    │
├──────────┴──────────┤
│ ROAS: 3.2 | CPA: ¥4,500 │
└─────────────────────┘
```

### フォントサイズの目安

| 要素 | PC用 | スマホ用 |
|---|---|---|
| スコアカードの数値 | 24-32px | 28-36px |
| グラフのタイトル | 14-16px | 16-18px |
| テーブルのセル | 12px | 14px |
| 凡例 | 10-12px | 非表示（スペース節約） |

### テーブルのカラム数を制限する

スマホでは横幅が狭いため、テーブルのカラム数は3〜4列に制限してください。PC用のテーブルが8列あっても、スマホ用には重要な列だけを選びます。

```
PC用テーブル:
日付 | チャネル | セッション | ユーザー | PV | 売上 | CVR | ROAS

スマホ用テーブル:
チャネル | 売上 | CVR
```

### 折れ線グラフの最適化

折れ線グラフをスマホに表示する場合のポイントです。

- X軸のラベル間隔を広めに設定（日付が重ならないように）
- 系列数は2〜3本まで（多すぎると判別できない）
- 凡例はグラフの上に配置（下だとスクロールが必要になる場合がある）

## BigQuery側でスマホ用のデータを準備する

スマホ用ダッシュボードは情報量を絞るため、BigQuery側でも簡潔なビューを用意すると効率的です。

```sql
CREATE OR REPLACE VIEW `project.dataset.mobile_dashboard_daily` AS
SELECT
  PARSE_DATE('%Y%m%d', event_date) AS date,
  COUNT(DISTINCT CONCAT(
    user_pseudo_id,
    CAST((SELECT value.int_value FROM UNNEST(event_params) WHERE key = 'ga_session_id') AS STRING)
  )) AS sessions,
  COUNT(DISTINCT user_pseudo_id) AS users,
  COUNTIF(event_name = 'purchase') AS purchases,
  IFNULL(SUM(ecommerce.purchase_revenue), 0) AS revenue,
  SAFE_DIVIDE(
    COUNTIF(event_name = 'purchase'),
    COUNT(DISTINCT CONCAT(
      user_pseudo_id,
      CAST((SELECT value.int_value FROM UNNEST(event_params) WHERE key = 'ga_session_id') AS STRING)
    ))
  ) AS cvr
FROM
  `project.analytics_XXXXXXX.events_*`
WHERE
  _TABLE_SUFFIX >= FORMAT_DATE('%Y%m%d', DATE_SUB(CURRENT_DATE(), INTERVAL 30 DAY))
GROUP BY
  date
ORDER BY
  date DESC
```

このビューは、日別の主要KPIだけを返すシンプルな構造です。カラム数が少ないため、Looker Studioのレンダリングも速くなります。

## フィルタの最適化

### ドロップダウンフィルタを使う

スマホではプルダウン型のフィルタが操作しやすいです。チェックボックス型やスライダー型は、指での操作が困難な場合があります。

### フィルタの数を減らす

PC用ダッシュボードでは5つ以上のフィルタを並べることもありますが、スマホ用では2〜3個に絞ります。

推奨フィルタ:
1. 日付範囲（期間選択）
2. デバイス（PC/スマホ/タブレット）
3. チャネル（主要なもののみ）

## パフォーマンスの考慮

スマホはPCと比べて通信速度やメモリが制限される場合があります。以下の点を意識してください。

- **データソースの数を減らす**: 1ページあたり1〜2データソースまで
- **BigQueryのキャッシュを長めに設定**: スマホからの閲覧は1日1〜2回が多いため、12〜24時間のキャッシュが適切
- **画像やロゴの容量を抑える**: 背景画像やロゴは軽量なものを使用

## まとめ

Looker Studioのスマホ最適化は、以下の手順で進めるのが効率的です。

1. **PC用ページとスマホ用ページを分ける**: キャンバスサイズ414pxの縦長ページを追加
2. **スコアカード中心に設計する**: 数値が大きく見やすいレイアウト
3. **情報量を絞る**: テーブルのカラムは3〜4列、フィルタは2〜3個
4. **BigQuery側も最適化する**: スマホ用の軽量ビューを別途用意
5. **URLを共有する**: スマホページ直リンクをブックマークに登録
