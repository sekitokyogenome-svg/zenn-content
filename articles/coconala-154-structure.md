## 154件分のデータを眺めると見えてくるもの

[前回の記事](https://zenn.dev/web_benriya/articles/coconala-154-3hours) で、ココナラの取引履歴154件を Playwright で全件構造化した話を書いた。

今回は、そのデータを眺めながら気づいた、**コンサル取引の数字構造**について書いていく。
ここでいう「構造」は、

- 1案件あたりの会話量
- 取引が完了するまでの期間
- リピート受注の割合
- ステータス遷移の典型パターン

のような、定量で見える部分のこと。

実装ログの読後感として「ふーん、で？」と思った方向けに、**構造化した先に何があったか**を書く回。

## 取引総量から見える、コンサル仕事の密度

154件すべての取引を構造化した結果、見えた基本指標は以下のとおり。

| 指標 | 値 |
|---|---|
| 取引総数 | 154件 |
| ユニーク顧客 | 98社 |
| 法人比率 | 33.8% |
| 平均取引期間 | 9.4日 |
| 総メッセージ数 | 2,043通 |
| 1案件あたりの平均メッセージ数 | 13.3通 |
| リピート受注 | 56件（36.4%） |

このうち、自分が一番気にしているのは **「1案件あたり13.3通のメッセージ」** という数字。

13回もメッセージを交わしているということは、コンサル仕事は実装そのものよりも**対話の比重が大きい**ということを意味している。
一見「実装力」で評価されそうな仕事でも、実態は「会話の密度ゲーム」だった、というのが構造化して初めて見えた気づきだ。

## 9.4日 × 13.3通の中身

平均9.4日で13.3通のやり取り。1日あたり約1.4通のペースになる。

ペース感としては、

- Day 1: 初回挨拶 + 相談内容の確認
- Day 2–3: 要件詳細のすり合わせ
- Day 4–7: 実装期間中の進捗報告と質問対応
- Day 8–9: 納品確認・微修正

という流れに、合計13通くらいで収まる計算だ。

これが「ココナラのプラチナランクは丁寧なヒアリングで取れる」と言われる根拠でもある気がしている。実装の速度より、**対話の密度が評価対象**になる。

## リピート受注36.4% のインプリケーション

98社のお客様で154件の取引 → 差分の56件はリピート。
取引のうち**3件に1件以上が、過去のお客様からの再依頼**ということになる。

これを別の言い方にすると、

- 1件きりで完結する顧客は約64%
- 36%の顧客は2回以上ご依頼いただいている

「3割の顧客が複数回」と聞くと薄く感じるかもしれないが、ストック型 SaaS でもなく、毎回完結する単発契約でこの数字は、自分としては悪くない数字な気がしている。

リピートを前提にサービス設計するなら、**納品時に「次の課題」が見えるドキュメントを残す**のが効果的だと思っている。
GA4 の計測整備で終わらせず、「次は BigQuery 連携で◯◯が見られるようになります」のような自然な次の一歩を提示しておくと、お客様側も2回目を依頼しやすくなる。

## status_tab の分布から見えるリスク

154件のステータス分布は次のとおり。

| ステータス | 件数 | 割合 |
|---|---|---|
| 完了 | 151件 | 98.0% |
| キャンセル | 2件 | 1.3% |
| 取引中 | 1件 | 0.7% |

完了率98% は高そうに見えるが、裏返せばキャンセルが2件ある。
7年で2件は頻度として低いものの、こうしたケースは振り返ると共通点があるはずだ、と仮説を持っている（ちゃんと言語化できていないので、Phase 4 の分析で詰めたい）。

経験則として、キャンセルが起きるのは概ね**初回ヒアリングで詰めきれなかった案件**だった気がしている。だからこそ、毎回ヒアリングを丁寧にやる必要がある — というのが、154件のデータを眺めたあとの再認識だ。

## 数字を構造化することの意味

ここまでの数字、ひとつひとつは取引履歴を見れば「なんとなく」わかる気はしていた。
ただ、構造化して JSON にしてみないと、

- 平均13.3通という具体的な数値
- リピート率36.4% という比率
- ステータスごとの割合

を**他人に説明できる形にはならなかった**。
ここが、自分の中で「データ化してよかった」と思っている一番のポイントだ。

数字は、サービスの「鏡」になる。
そして鏡があると、サービス改善の議論が**感覚論から数字を起点にした議論**に変わる。

## まとめ：コンサル仕事は「会話の密度ゲーム」

154件のデータから見えた構造をまとめると、

- 1案件あたり平均9.4日で13.3通のやり取り（=会話の密度が高い）
- 顧客の36% はリピート（= 関係構築型のビジネス）
- 完了率98%、キャンセルは初回ヒアリング不足が原因（仮説）

コンサル仕事は実装よりも**対話の量と質**で決まる側面が大きい、というのが154件の数字が語っていることだ。

自分としては、フリーランスでデータ仕事を続けるなら、**自分の取引履歴をデータ化して定期的に振り返るのが、サービス改善の最短ルート**な気がしている。

読者のみなさんも、自分のサービスの「会話の密度」、計測してみたことありますか？
取引履歴をデータ化するだけで、見えるものは思った以上に多い気がしています。
